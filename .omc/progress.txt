## Key learnings to carry forward

- Cloudflare Containers don't auto-restart on image refresh — must bump idFromName() suffix
- Cloudflare Containers max_instances quota — multiple bumped DO instances compete
- Worker secrets must be EXPLICITLY forwarded to Container via DurableObject.start({env}) — they don't auto-propagate
- gpt-5 family models reject max_tokens, require max_completion_tokens + temperature=1
- alibaba/* prefix not supported by Unified Billing; use workers-ai/@cf/qwen/* if needed (we dropped Qwen)
- Reasoning models (gemini-2.5-pro, QwQ) consume internal tokens; need 4096+ budget for visible output
- Gemini Flash moderator output often gets truncated mid-JSON at 8192 budget — _repairTruncatedJson is REQUIRED
- Worker postgres client: getSql(env) creates per-request, never module-level (I/O context binding errors)
